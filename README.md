## 🎯 产品简介

**SoundNet** 是一套**控制与播放分离**的分布式频道音乐播放系统。控制页面与播放器完全解耦，通过 Go 后端 + WebSocket 实现实时双向通信。

### 设计理念

传统的音乐播放器通常是"一个界面控制一个播放器"。SoundNet 打破这一限制——**一个控制页面可同时指挥多个播放器**，且播放器遍布 Web / Android / Windows / Linux 全平台。

### 适用场景

- 🏠 **多房间音乐系统** — 不同房间不同频道，独立控制
- 🏢 **商业背景音乐** — 统一调度多个播放终端
- 🎧 **个人多设备** — 电脑控制，手机/平板播放
- ⏰ **定时音乐任务** — 闹钟唤醒、睡眠定时关闭

## ✨ 核心功能

### 📡 控制播放分离

控制页面与播放器完全解耦，通过 Go 后端 WebSocket 调度。一个控制页面可同时指挥多个播放器。

### 📻 频道管理体系

频道是播放器逻辑集合。创建频道、分配播放器、独立队列。同一频道内的所有播放器同步接收指令。

### 🎵 智能播放队列

Redis 持久化队列，支持添加、删除、拖拽排序、清空。实时推送队列变更到控制端和播放器。

### 🔍 全网音乐搜索

接入第三方音乐服务 API，搜索歌曲、歌单，获取播放链接和歌词。预留接口适配层，可切换数据源。

### ⏰ 闹钟 & 睡眠定时

频道级闹钟系统：渐入渐出、冲突策略、指定歌曲播放。睡眠定时自动暂停/停止，闭环体验。

### 🎛️ 精细播放控制

播放/暂停、上一曲/下一曲、进度拖拽、音量调节（统一或单独）。实时状态同步，所见即所得。

## 🏗️ 系统架构

```
┌──────────────┐     WebSocket      ┌──────────────┐     WebSocket      ┌──────────────┐
│              │  ◄──────────────►  │              │  ◄──────────────►  │              │
│  Vue 3 控制   │                    │  Go API 平台   │                    │  播放器客户端  │
│   控制页面     │                    │   指令调度     │                    │  Web / Android│
│              │                    │              │                    │  Windows/Linux│
└──────────────┘                    └──────┬───────┘                    └──────────────┘
                                           │
                            ┌──────────────┼──────────────┐
                            │              │              │
                     ┌──────┴─────┐  ┌──────┴─────┐  ┌────┴─────┐
                     │   Redis    │  │   SQLite   │  │第三方 API │
                     │  播放队列   │  │ 播放器信息   │  │ 音乐数据源 │
                     └────────────┘  └────────────┘  └──────────┘
```

### 数据流说明

1. **用户**在 Vue 控制页面操作（选择歌曲、调节音量等）
2. 操作指令通过 **WebSocket** 发送到 Go 后端
3. Go 后端处理业务逻辑（检索 Redis 队列、更新 SQLite 状态等）
4. 指令通过 WebSocket **推送到目标播放器**执行
5. 播放器状态变化**实时回传**到控制页面

所有通信基于 WebSocket，**实时双向低延迟**。

## 🛠️ 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| **前端** | Vue 3 + TypeScript + Vite | 响应式控制页面 |
| **后端** | Go + Gorilla WebSocket | HTTP 服务、WebSocket 指令调度 |
| **缓存** | Redis | 播放队列持久化 |
| **存储** | SQLite | 播放器信息、频道配置 |
| **运维** | Docker Compose | 一键部署 api / redis / vue 三服务 |
| **图标** | Lucide Icons | 统一图标风格 |

## ❓ 常见问题

**Q: 控制页面和播放器是否必须在同一网络？**  
A: 不需要。只要播放器能连接到 Go 后端 WebSocket 服务即可，支持跨网络部署。

**Q: 一个频道可以绑定多少个播放器？**  
A: 无数量限制。频道内所有播放器同步接收指令。

**Q: 支持哪些音乐平台？**  
A: 通过接口适配层可切换数据源，默认支持多个主流音乐平台 API。

**Q: 播放器客户端如何获取？**  
A: Web 端直接访问浏览器打开；Android / Windows / Linux 客户端需下载安装。
